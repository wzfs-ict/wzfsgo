ScratchJr + AI Thinking
In Grade 5 you use all four ScratchJr concepts — Sequence, Loop, Event, Trigger — and explore what makes Artificial Intelligence (AI) different from the code you write.
ScratchJr — Grade 5 Level Skills
Grade 5 Skills

What You Can Build Now

  • ⚙️ Multi-page interactive stories
  • ⚙️ Parallel scripts (many things at once)
  • ⚙️ Enemy characters with patrol loops
  • ⚙️ Decision-based events (bump → reset)

Algorithm

What Makes an Algorithm?

An algorithm is a clear, step-by-step set of instructions that solves a problem. Every ScratchJr script is an algorithm — it has a start, clear steps, and a predictable result.

A good algorithm is: precise (no guessing), finite (has an end), and unambiguous (only one meaning).

The Big Idea — Code vs AI
Feature Computer Programming (Your Code) Artificial Intelligence (AI)
How does it work? You write every rule yourself. The computer follows exactly what you say. The AI looks at thousands of examples and finds patterns by itself.
Who decides? You do — you write every instruction. The AI figures it out from data, not from rules you write.
Is it predictable? Yes — the same input always gives the same output. Not always — AI can give different answers to similar questions.
Can it learn? No — your ScratchJr code only does what you coded. Yes — AI improves as it sees more examples.
Example "IF bump → go home" — you wrote that exact rule. A face recognition app — no one wrote rules for every face, it learned from millions of photos.
Key Concept

Selection (Decision)

Selection means a program can choose what to do based on what happens. In ScratchJr this appears as event-based triggers: "IF the hero bumps the enemy, THEN go home."

Key word: Conditional — "if this condition is true, do that action."

Key Concept

Concurrent Programming

Concurrent means two things happening at the same time. In ScratchJr, your hero moves while an enemy patrols — two separate scripts running simultaneously. This is how real games work!

Algorithm
A precise, step-by-step set of instructions that solves a specific problem.
Sequence
Steps that run one after another in order.
Loop
A block or set of blocks that repeats automatically.
Selection
The program chooses what to do based on a condition (IF something happens, THEN do this).
Event
Something that happens — a tap, bump, green flag press, or message received.
Trigger
A yellow block that watches for an event and starts a script running.
Concurrent
Two or more things running at the same time.
Artificial Intelligence (AI)
A computer system that learns from examples and data — it finds patterns instead of following rules you write.
Debugging
Finding and fixing mistakes (bugs) in your code.
Conditional
"If [something], then [do this]" — a statement that depends on a condition being true.
🧠
Assessment Tip — AI Question: The key difference is how the rules are created. In your code, you write the rules. In AI, the system finds the rules from data. You may be asked: "Can your ScratchJr project learn?" — the answer is No, because it only does exactly what you coded.

⭐ Remember for the Assessment

  • An algorithm is precise, finite, and unambiguous — like a recipe.
  • Your ScratchJr code uses Sequence + Loop + Selection (Events).
  • AI learns from examples. Your code follows rules you write. They are fundamentally different.
  • Concurrent = two things running at the same time.
  • Selection in ScratchJr = event triggers responding to what happens (bump, tap).

Botley — Physical Computing
Botley is a programmable robot you program without a screen — using physical buttons and sequences. Physical computing means using code to control real-world hardware.
What is Physical Computing?
Key Idea

Code Controls Hardware

Physical computing means writing code that controls a real physical device — not just a character on a screen. Botley is a real robot that moves around based on the steps you program into it.

Key Idea

Screen-Free Coding

Botley uses screen-free coding — you program it using the buttons on its back, not a tablet or computer. This helps you understand that coding is about logical sequences — not just apps!

How to Program Botley
Step by Step

Programming Process

  • 1️⃣ Press the Home button once to wake Botley
  • 2️⃣ Press your movement buttons in order
  • 3️⃣ Each press = one step stored in memory
  • 4️⃣ Press the Go button to run the sequence
  • 5️⃣ Watch Botley move! Adjust if needed.
Example Program

Navigate a Simple Path

To move Botley: forward 2, turn right, forward 1:

⬆️ Fwd ⬆️ Fwd ➡️ Right ⬆️ Fwd ▶️ Go!
Botley Vocabulary
Physical Computing
Using code to control real-world hardware — like robots, motors, lights, and sensors.
Screen-Free Coding
Programming a device using buttons or physical controls instead of a computer screen.
Sequence (Botley)
The order of movement commands you press — Botley follows them exactly in that order.
Step
One single move — one press of forward moves Botley exactly one grid square forward.
90° Turn
A quarter turn — turning left or right once rotates Botley 90 degrees to face a new direction.
Debug (Botley)
If Botley goes the wrong way, you check your sequence of presses — find the mistake and re-enter correctly.
Input
The commands you press into Botley — what you give to the system.
Output
What Botley does — the physical movements that result from your input.
💡
Assessment Tip: Direction matters! If Botley is facing right and you press Turn Left, it will now face upward (north), not left of the screen. Think about which direction Botley is currently facing before pressing a turn button. Draw a grid and trace the path on paper first!
Real World Link

Where is Physical Computing Used?

  • 🚗 Self-driving cars
  • 🦾 Factory robots building things
  • 💡 Smart home lights that turn on automatically
  • 🚦 Traffic lights that change based on sensors
  • 🏥 Medical devices that monitor patients
Decomposition

Breaking Down the Problem

Decomposition means breaking a big problem into smaller steps. With Botley, you look at the whole path and split it into individual moves. "Go to the end of the maze" becomes: Forward, Forward, Right, Forward, Left…

⭐ Remember for the Assessment

  • Physical computing = code that controls real hardware.
  • Botley is screen-free — programmed with physical buttons.
  • Each button press = one stored step. The sequence runs when you press Go.
  • Botley turns in 90° increments — always relative to the direction it's currently facing.
  • Input = what you press. Output = what Botley does.
  • Debugging Botley means finding which step in the sequence was wrong.

3D Modelling Basics
Design three-dimensional objects using digital tools. In Grade 5 you understand not just how to build, but why design decisions are made.
Core 3D Concepts
3D Model
A digital object that has width (X), height (Y), and depth (Z) — it exists in three dimensions.
X, Y, Z Axes
X = left/right, Y = up/down, Z = forward/backward. All 3D positions are described using these three axes.
Vertex (vertices)
A single point (corner) in a 3D model. Multiple vertices connected form edges and faces.
Edge
A straight line between two vertices — the outline of a face.
Face
A flat surface bounded by edges — the "skin" of a 3D object. A cube has 6 faces.
Mesh
The complete 3D shape made of all its vertices, edges, and faces together.
Scale / Resize
Make a shape bigger or smaller — either uniformly (all axes the same) or non-uniformly (stretch on one axis).
Boolean Operation
Combining shapes using Add (Union), Subtract (Difference), or Intersect. The Hole tool in Makers Empire uses Boolean Subtract.
3D Printing
Physically producing a 3D model by building it layer by layer from a material like plastic.
Prototype
An early version of a design — used for testing before making the final version.
Tolerance
The small gap left between parts that need to fit together when 3D printed — because printing isn't 100% precise.
Design Process (Grade 5 Level)
Process

Empathy-First Design

Grade 5 designers think about the user's needs first — not just what looks cool. Ask: Who will use this? What do they struggle with? How can my design help them?

Process

Iteration

Iteration means improving your design through testing and feedback — making version 1, testing it, improving to version 2, testing again. Good designers rarely get it right on the first try!

Process

Form vs Function

Form = how it looks. Function = what it does. The best designs balance both — they look good AND work well. A chair that's beautiful but uncomfortable fails at function.

🎯
Assessment Tip: Know the difference between Vertex, Edge, and Face — these basic 3D geometry terms are commonly tested. A cube has: 8 vertices, 12 edges, 6 faces. A triangular prism has 6 vertices, 9 edges, 5 faces.

⭐ Remember for the Assessment

  • 3D objects have X (width), Y (height), Z (depth) axes.
  • A mesh is made of vertices (points), edges (lines), and faces (flat surfaces).
  • Boolean operations: Add shapes together or Subtract (cut a hole).
  • Cube: 8 vertices, 12 edges, 6 faces.
  • Form = how it looks. Function = what it does.
  • Iteration = improving through testing — designers do this many times.

Vector Graphics
Create images using mathematical shapes and paths — not pixels! Vector images can be made any size without becoming blurry.
Open method.ac Editor ↗
Raster vs Vector — The Big Difference
Feature 🖼️ Raster (e.g. JPEG, PNG, Pixel art) 🖊️ Vector (e.g. SVG)
Made of Pixels (tiny coloured squares) Mathematical paths, points, and shapes
When zoomed in Becomes blurry / pixelated — loses quality Stays perfectly sharp at any size
File size Can be large (stores every pixel's colour) Usually small (stores mathematical descriptions)
Best for Photos, detailed paintings, screenshots Logos, icons, illustrations, print designs
Examples Photos on Instagram, Piskel animations Company logos, app icons, maps
Vector Graphics Vocabulary
Vector
An image made of mathematical shapes and paths — not pixels. Can be resized without losing quality.
Path
A line or shape defined by a start point, end point, and curve information. The building block of vector art.
Anchor Point
A point on a path that defines a corner or curve. Moving anchor points changes the shape of the path.
Stroke
The visible outline of a path or shape — you can change its colour, thickness (width), and style.
Fill
The colour inside a closed shape. A circle has a fill (the inside colour) and a stroke (the outline colour).
Node
Another word for an anchor point — a control point on a path that you can move to reshape the line.
Bezier Curve
A smooth, mathematically defined curve used in vector graphics. Created by dragging handles on anchor points.
SVG
Scalable Vector Graphic — a file format for vector images. Used widely on websites and in apps.
Layer
A level in a design — objects on higher layers appear in front of objects on lower layers.
Group
Combine multiple shapes so they can be moved and edited together as one object.
Boolean (Vector)
Combine two shapes by: Union (merge), Subtract (cut one from the other), or Intersect (keep only the overlap).
Scalable
"Scalable" in SVG means the image looks perfect at any size — from a tiny icon to a giant billboard.
method.ac Editor — Key Tools
Tool

Selection Tool (V)

Click and drag to select objects. Move them, resize them, or delete them. The most basic tool — you use it constantly.

Tool

Pen / Path Tool (P)

Click to create straight-line paths. Click and drag to create curved (Bezier) paths. Each click creates a new anchor point.

Tool

Shape Tools (R, E)

Draw rectangles (R) and ellipses/circles (E) directly. Hold Shift to make perfect squares or perfect circles.

Tool

Node / Anchor Edit (N)

Select individual anchor points on a path and move them to reshape curves and lines.

Properties

Fill & Stroke Panel

Choose the inside colour (fill) and outline colour and thickness (stroke) for any selected shape.

Properties

Z-Order (Arrange)

Bring shapes to the front or send them behind others. Like stacking pieces of paper — the top piece covers the one below.

🖊️
Assessment Tip: The most common exam question about vector graphics is about scalability — "Why would a designer choose vector over raster for a logo?" Answer: Because a vector logo stays crisp at any size, from a business card to a billboard. Raster images become blurry when enlarged.

⭐ Remember for the Assessment

  • Vector images use mathematical paths, not pixels.
  • Vectors are scalable — they look sharp at any size. Raster images pixelate when enlarged.
  • Stroke = outline. Fill = inside colour.
  • Anchor points (nodes) define the shape of paths.
  • SVG = Scalable Vector Graphic — the main vector file format.
  • Logos, icons, and illustrations are best made as vectors. Photos are raster.

Sequence
Steps that run one after another in a specific order — the core of every algorithm.
Sequence
A series of steps that execute one at a time, in a specific order.
Algorithm
A complete set of instructions to solve a problem — always has a clear start, ordered steps, and a defined end.
Parallel Execution
Two or more sequences running at the same time (e.g. hero moving while enemy patrols).
In ScratchJr
▶ Flag → → Move → ↑ Jump → 📤 Send msg
In Botley

Each button press adds one step to the sequence. Botley runs them all in order when you press Go.

⬆️ Fwd ⬆️ Fwd ➡️ Right ⬆️ Fwd ▶️ Go!

⭐ Key Facts — Sequence

  • Steps run one at a time, in order — changing order changes the result.
  • Applies to both ScratchJr (blocks) and Botley (button presses).
  • Parallel execution = two sequences running at the same time.
🔄
Loop
Repeat blocks or commands automatically — the smart way to avoid writing the same step many times.
Repeat Forever

Infinite Loop

Runs until stopped. Used for enemy patrols, spinning decorations, background animations. Put at the END of a script.

▶ Flag→♾ Forever [ Patrol ]
Repeat N Times

Finite Loop

Runs a set number of times, then the script continues. Used for exactly 5 jumps, flashing 3 times, etc.

▶ Flag→🔁 ×5 [ Jump ]→♪ Pop
Loop
A block or group of blocks that repeats automatically.
Iteration
One single run of the loop. 5 iterations = the loop ran 5 times.
Infinite Loop
A loop that never ends on its own (Repeat Forever).

⭐ Key Facts — Loop

  • Loops repeat code automatically — shorter, smarter scripts.
  • Repeat Forever: never stops. Repeat N: stops after N runs.
  • In the Phantom Maze — enemy uses Repeat Forever to patrol. Player resets with Go Home on bump.
⚡
Event & Trigger
Events are things that happen. Triggers are yellow blocks that detect events and start your code — the "IF this, THEN that" of ScratchJr.
Event
Something that happens in the project: flag tapped, sprite tapped, bump, message received.
Trigger
A yellow block that watches for an event and starts the attached script when it occurs.
Green Flag ▶
Starts all scripts with this trigger when the project begins.
Start on Tap
Triggers when the player taps (touches) that sprite.
Start on Bump
Triggers when this sprite collides with another sprite.
Receive Message
Triggers when a matching coloured message is sent.
Selection
The concept of "IF [event], THEN [action]" — choosing different actions based on what happens.
🔑
Grade 5 key point: Triggers are how ScratchJr implements Selection — the "IF/THEN" concept. When you use Start on Bump → Go Home, you're saying: IF the character bumps something, THEN go home. That's a conditional decision.

⭐ Key Facts — Event & Trigger

  • Every script must start with a trigger (yellow block) — no trigger = code never runs.
  • Trigger types: Flag, Tap, Bump, Message — know all four!
  • Messages must match by colour.
  • Triggers implement Selection — IF event happens, THEN script runs.
