Piskel — Stop Motion & Animation
Learn how animation works using frames, and create your own pixel art animations with Piskel.
What is Stop Motion?
Key Idea

Fooling Your Eyes

Stop motion animation works by showing many still images (frames) one after another very quickly. Your brain sees them as smooth movement — this is called the persistence of vision.

Key Idea

Frames Per Second (FPS)

FPS means how many frames are shown every second. More FPS = smoother movement. Fewer FPS = choppy, jumpy movement. Most cartoons use 12–24 FPS.

Key Idea

The Onion Skin Tool

In Piskel, the Onion Skin shows a faded ghost of the previous frame while you draw the next one. This helps you place the new frame correctly so movement looks smooth.

Each frame is a slightly different image. Shown quickly in sequence, they create animation:

Piskel Tools & Features
Frame
A single still image. Many frames played in sequence create an animation.
Canvas
The drawing area in Piskel — where you draw each frame of your animation.
Pencil Tool
The main drawing tool — draws pixel by pixel on the canvas.
Fill Tool (Bucket)
Fills a closed area with a colour in one click — much faster than colouring pixel by pixel.
Eraser Tool
Removes colour from pixels — makes them transparent (invisible).
Onion Skin
Shows a ghost image of the previous frame to help you draw the next frame in the right position.
FPS (Frames Per Second)
How fast the frames play. Higher FPS = smoother animation. Lower FPS = choppier, slower look.
Duplicate Frame
Makes a copy of the current frame — useful so you only need to change a small part instead of redrawing everything.
Export / Download
Save your animation as a GIF (moving image) or PNG (still image) to use it outside Piskel.
GIF
A file type that stores a short, looping animation. Your exported Piskel animation is saved as a GIF.
Pixel Art
Art made by placing individual coloured squares (pixels) carefully. It has a retro, blocky look — like old video games.
Sprite Sheet
A single image that contains all the frames of an animation laid out in a grid — used in games and apps.
💡
Assessment Tip: Know the difference between FPS (speed of animation) and number of frames (how long the animation is). A 10-frame animation at 5 FPS plays for 2 seconds. A 10-frame animation at 10 FPS plays for 1 second — faster!
Animation Principles to Know
Principle

Persistence of Vision

Your brain holds onto an image for a tiny moment after it disappears. When frames change quickly, your brain "fills in" the gaps and sees smooth movement.

Principle

Squash & Stretch

A ball flattens when it hits the ground (squash) and stretches when it bounces up (stretch). This makes animation feel alive and bouncy.

Principle

Anticipation

A small movement before a big action makes it look more realistic. Example: a character crouches down slightly before jumping up.

⭐ Remember for the Assessment

  • Stop motion works by showing many still frames very quickly.
  • FPS = Frames Per Second. More FPS = smoother movement.
  • The Onion Skin tool shows the previous frame as a ghost to guide drawing.
  • Use Duplicate Frame to copy a frame and only change what moves.
  • Export as a GIF to share your animation.
  • Pixel art is made by placing individual squares (pixels) to create images.

In ScratchJr, a Page is like a different scene or level in your project. You can have many pages and send the player from one to another using code.

A sprite can be made invisible using Hide. A hidden sprite can still run code! This is used for "controller" sprites that manage events behind the scenes.

Timing — The Wait Block

The Wait block pauses the script for a set number of tenths of a second before the next block runs. This helps synchronise multiple sprites.

▶ Green Flag → ⏱ Wait × 3 → → Move Right → ♪ Pop

The sprite waits 0.3 seconds, then moves right, then plays a sound.

💡
Assessment Tip: If two sprites need to do things at the same time in sequence, use Send & Receive Message to coordinate them — Sprite A finishes its action and then sends a message; Sprite B receives it and starts its action.

⭐ Grade 4 ScratchJr — Remember

  • Multiple sprites can each have their own scripts running at the same time.
  • A Page is a scene — use the Go to Page block to switch scenes.
  • Hidden sprites can still run code — they're useful as invisible controllers.
  • Wait blocks add a pause — useful for timing effects.
  • Messages coordinate two different sprites — send colour must match receive colour.
➡️
Sequence
Steps that run one after another, in order. The foundation of all programming.
Sequence — Quick Review
Definition

In Order, One at a Time

A sequence is a set of instructions that execute one after another. Each step must finish before the next begins. In ScratchJr, blocks connect left-to-right and run in that order.

Grade 4 Focus

Sequence Across Sprites

Each sprite has its own sequence of blocks. Two sprites can run their own sequences at the same time — called parallel execution. They don't wait for each other.

Example — Two Parallel Sequences
SPRITE 1 — runs at the same time as Sprite 2
▶ Flag → → Move → ↑ Jump
SPRITE 2 — runs at the same time as Sprite 1
▶ Flag → ♾ Forever → 🔄 Spin
Sequence
Steps that run one at a time, in a specific order from start to finish.
Parallel Execution
Two or more sprites running their own sequences at the same time.
Wait Block
Pauses the sequence for a set time before the next block runs.

⭐ Key Facts — Sequence

  • Each block in a sequence runs after the previous one finishes.
  • Changing the order of blocks changes the result.
  • Multiple sprites can run their own sequences at the same time.
  • Use Wait to pause a sequence and control timing.
🔄
Loop
Makes part of a script repeat automatically — saving time and making code smarter.
Loop — Quick Review
Repeat Forever

Runs Until Stopped

Never stops on its own. Used for background effects, enemies patrolling, decorative animations. Must be at the end of a script or it will block everything after it.

▶ Flag → ♾ Forever [ 🔄 Spin ]
Repeat N Times

Runs a Set Number of Times

Runs the inside blocks exactly N times, then the script continues. Used for things like jumping 3 times, flashing 5 times, or moving a set distance.

▶ Flag → 🔁 ×4 [ → Move ] → ♪ Pop
Loop
A block (or set of blocks) that repeats automatically.
Iteration
One pass through the loop. "5 iterations" = the loop ran 5 times.
Infinite Loop
A loop that never ends — like Repeat Forever. Stops only when the program is stopped.
⚠️
Common Mistake: A Repeat Forever block placed in the middle of a script will trap the code — any blocks AFTER the forever loop will never run. Put forever loops at the end of a script.

⭐ Key Facts — Loop

  • Loops repeat code automatically — no need to write the same block over and over.
  • Repeat Forever — never stops. Repeat N Times — stops after N runs.
  • Each run of the loop is called an iteration.
  • Forever loops should come at the END of a script.

Event & Trigger
Events are things that happen. Triggers are yellow blocks that detect events and start scripts running.
Events & Triggers — Grade 4 Level
Event
Something that happens in the project — a tap, a bump, a message being received, the green flag being pressed.
Trigger
A yellow block at the start of a script that watches for a specific event and starts the script when it occurs.
Green Flag ▶
Triggers all scripts with this block when the project starts (flag is tapped).
Start on Tap
Triggers when the player taps that specific sprite.
Start on Bump
Triggers when this sprite physically touches another sprite on the stage.
Receive Message
Triggers when a matching coloured message is sent from another sprite.
Send Message
Sends a coloured message that triggers any sprite waiting with a matching Receive Message block.
Grade 4 Example — Coordinated Scene

A character completes a path, sends a message, and a second character reacts:

HERO — arrives at the castle
▶ Flag → →→ Move → 📤 Send 🔵
CASTLE DOOR — opens when hero arrives
📩 Receive 🔵 → 📈 Grow ×3
🔑
Key rule: Message colours must MATCH. A blue send only triggers a blue receive. If the colour doesn't match, nothing happens — this is a very common mistake to check for in the assessment!

⭐ Key Facts — Event & Trigger

  • Events are things that happen (tap, bump, message received).
  • Triggers are yellow blocks that watch for events and start scripts.
  • Without a trigger, a script will NEVER run.
  • Messages coordinate different sprites — but colours must match.
  • One send message can trigger multiple sprites if they all have matching receive blocks.
