Topic 1

Data Modelling with Spreadsheets

Core Vocabulary
Spreadsheet Components
Cell — The single box where data lives, identified by column letter + row number (e.g. C3).
Column — A vertical set of cells, labelled A, B, C …
Row — A horizontal set of cells, labelled 1, 2, 3 …
Range — A group of cells from one corner to another, written as A1:C5.
Fill Handle — The small square at the bottom-right of a selected cell. Drag it to copy a formula down or across.
Data vs. Information
Data — Raw facts or figures with no context (e.g. the numbers 150, 75, 125).
Information — Data that has been organised, labelled, and given context so it carries meaning (e.g. a chart titled "Monthly Sales").
💡 A spreadsheet turns data into information through formulas, sorting, and charts.
Formulas & Operators
Writing Formulas — Always Start with =

Every formula must begin with an equals sign. Use cell references (e.g. =A1+B1), not typed numbers — that way if the data changes, the result updates automatically.

OperatorSymbolExampleResult
Addition+=C2+D2Adds the values in C2 and D2
Subtraction-=C2-D2Subtracts D2 from C2
Multiplication*=C2*D2Multiplies C2 by D2
Division/=C2/D2Divides C2 by D2
Absolute vs. Relative Cell References
Relative — =A1+B1
Changes when you copy the formula to another cell. The reference shifts with it.
Absolute — =$A$1+B1
The $ locks that part of the address. It stays fixed even when copied.
💡 Use absolute references when one cell (like a tax rate) must stay the same across many formulas. Add $ in front of the column letter, the row number, or both.

Worked Example — Party Shopping List
Step-by-Step Problem

You have a table: Item | Price per pack | Quantity | Cost | Total Cost

1
In the Cost column for the first item (e.g. cell E2), type: =C2*D2 (Price × Quantity).
2
Use the Fill Handle to drag the formula down to E6 to fill in all items automatically.
3
In the Total Cost cell, type: =SUM(E2:E6) to add up all costs.
4
Select the Item column and Cost column, then Insert → Chart to visualise the data.
💡 The exam may ask you to write the formula for a specific cell. Always use cell references, never typed numbers.
Formatting & Data Tools
Formatting

Change fonts, text alignment, borders, and cell fill colour. Apply number formats: currency, percentage, date. Merge cells to create neat headings.

Sorting & Filtering

Sort data A→Z or 1→9 (or reverse). Do a multi-level sort: e.g. sort by class, then by score within each class. Filter to show only rows that match a condition.

Conditional Formatting

Automatically changes the appearance of a cell based on its value. Example: highlight all scores below 40 in red so low marks stand out instantly.

Topic 2

Programming Concepts with Scratch

Core Programming Concepts
Sequence

The order in which commands are executed. A computer follows instructions exactly as written — one after another, top to bottom.

Example: say "Hi" → wait 1s → say "Bye"
Selection (IF/THEN/ELSE)

Allows a program to make decisions. It checks a condition — if True, one set of blocks runs; if False, the other runs.

Scratch block: if <condition> then … else …
Decomposition

Breaking a complex problem into smaller, manageable subproblems. Each subproblem becomes easier to solve on its own.

Example: Break "Dance Game" into: move1, move2, move3
Variables
What is a Variable?

A variable is a named storage location in the computer's memory that holds a piece of data — like a labelled box. The value inside can change as the program runs.

Use variables to: keep a score, store a player's name, count how many times something happened, track lives remaining.
In Scratch:
set [score] to 0
change [score] by 1
Subroutines (My Blocks)
Custom Blocks in Scratch

A subroutine (called a My Block in Scratch) is a named sequence of instructions grouped together to perform a specific task. You define it once and call it whenever you need it.

Benefits:
  • Avoids repeating the same code
  • Makes code easier to read
  • Easier to find and fix bugs
  • Can accept parameters (inputs) so it works with different values
Example:
define ask_name
→ ask "What is your name?" and wait
→ set [name] to (answer)

Then call it anywhere: ask_name
Loops (Repetition)
Types of Loops in Scratch
Loop BlockWhat it doesWhen to use it
repeat (10)Runs its contents a fixed number of timesWhen you know exactly how many repeats you need
foreverRuns its contents non-stop until the program stopsMain game loop — checking for key presses, collisions
repeat until <condition>Keeps repeating until the condition becomes TrueWhen you don't know how many times in advance
Operators
Logical Operators — AND / OR
AND — Both conditions must be True for the result to be True.

Example: score > 50 AND lives > 0 → True only if both are correct.
OR — At least one condition must be True for the result to be True.

Example: key = "a" OR key = "left arrow" → True if either key is pressed.
💡 NOT reverses a Boolean: NOT(touching wall?) is True when the sprite is NOT touching a wall.
Worked Example — Reading Code
Code Analysis: how_are_you Subroutine

Study the block sequence below and trace what happens:

define how_are_you
ask "Are you OK?" and wait
set [health] to (answer)
if <health = "yes"> then
   say "Good, glad to hear it" for 2 secs
else
   say "Oh no, hope you feel better soon" for 2 secs
say "I have another question" for 2 secs
✅ If the user types "yes", the sprite says "Good, glad to hear it". It then always says "I have another question" regardless of the answer, because that block is outside the if/else.

Topic 3

3D Modelling with Tinkercad

The Digital Workbench
Key Interface Elements
The Workplane — The blue grid. It is the floor where you build. All shapes should sit on (or above) this surface.
View Cube — A cube in the corner of the screen. Click any face to jump to that camera angle (Top, Front, Side).
Perspective View — The default 3D view; objects appear to get smaller with distance (realistic).
Orthographic (Flat) View — Removes perspective. Use this to check shapes are perfectly touching or aligned.
Why Check Multiple Views?

A shape can look correct from the front but be floating above the workplane or misaligned from the side. Always check Top, Front, and Side before finishing.

⚠️ Only looking at one view is a common mistake. Shapes can be in the wrong position in 3D and you will not see it unless you rotate the camera.
Moving, Measuring, and Lifting
Precision Building — Never Guess
ActionHow to do itWhy it matters
Set exact dimensionsClick the white corner boxes that appear when a shape is selected. Type the width, length, and height in mm.Ensures your model is the correct size, not just "looks about right".
Lift a shape (Z-axis)Click and drag the small black cone above the shape upward.Raises the shape off the workplane so it can sit on top of another shape.
Rotate preciselyUse the curved arrows around the shape. Hold Shift while rotating to snap to exactly 45° increments.Keeps angles accurate; avoids shapes at odd, unintentional angles.
Overlapping shapes3D shapes can be placed inside or on top of each other — this is how holes and joins are made.Foundation of all carving and combining techniques.
Advanced Techniques
Making Holes — Grouping Solids and Holes
1
Place a smaller shape inside a bigger solid shape where you want the hole to be.
2
Select the smaller shape and click Hole in the Inspector panel. It turns grey/transparent to show it is now a hole cutter.
3
Select both the solid shape and the hole shape together (Shift+click or drag to select).
4
Press Ctrl+G (or click Group). The hole shape carves out the solid, creating a hollow space.
💡 The hole shape must overlap/be inside the solid to cut it. If it's outside, nothing is carved.
Duplicate — Ctrl+D

Copies the selected shape. It remembers the last direction and distance you moved the copy. This makes building stairs, fences, or repeating patterns very fast — just keep pressing Ctrl+D.

Scribble Tool

Lets you draw a custom 2D outline that Tinkercad then extrudes into a 3D shape. Use it when none of the standard shapes (boxes, cylinders, spheres) match what you need.

The Master's Check — Before You Submit
Ask Yourself These Four Questions
✅ Did I check the Top, Front, and Side views to confirm nothing is floating or misaligned?
✅ Did I type exact measurements (mm) instead of dragging and guessing?
✅ Are my shapes in the correct proportion compared to each other?
✅ Did I Group all my shapes into one final object at the end?

Topic 4

Computer Systems & AI (Data Science)

Computer Systems Fundamentals
Hardware vs. Software
Hardware — The physical parts of a computer you can touch: CPU, RAM, hard drive, screen, keyboard, mouse, GPU.
Software — Programs and data that run on the hardware: operating systems (Windows, macOS), apps, games.
Firmware — Software permanently stored in hardware (e.g. the BIOS chip that starts a computer).
The CPU — Brain of the Computer
CPU (Central Processing Unit) — Carries out all instructions. Speed is measured in GHz (gigahertz).
RAM (Random Access Memory) — Temporary working memory. Holds data and programs currently in use. Lost when the computer is turned off.
Storage (HDD / SSD) — Permanent storage. Keeps files even when powered off. SSD (Solid State Drive) is faster than HDD.
Input, Process, Output — The IPO Model
How Computers Process Data
⌨️
Input
Data enters the system.
Keyboard, mouse, microphone, scanner, sensors.
⚙️
Process
CPU performs calculations and applies instructions to the data.
🖥️
Output
Result is produced.
Screen, printer, speaker, actuator.
Storage is sometimes added as a fourth stage — data can be saved and retrieved during any part of the process.
Networks & the Internet
Network Types
LAN (Local Area Network) — Connects devices in a small area like a school or home.
WAN (Wide Area Network) — Connects devices across large distances. The Internet is the world's largest WAN.
Router — Directs data packets between networks. Your home router connects your devices to the Internet.
Key Internet Concepts
IP Address — A unique number identifying each device on a network (like a postal address).
DNS — Translates human-readable website names (google.com) into IP addresses.
HTTP / HTTPS — Protocols for sending web pages. HTTPS is encrypted and secure.
Data Science & Artificial Intelligence
What is Data Science?

Data Science is the process of collecting, cleaning, analysing, and visualising large sets of data to find patterns and make decisions. It combines maths, statistics, and computing.

Collect — Gather raw data from sensors, surveys, databases, or the web.
Clean — Remove errors, duplicates, and missing values so the data is accurate.
Analyse — Use statistics and tools (like spreadsheets) to find patterns and trends.
Artificial Intelligence (AI) Concepts
Machine Learning
AI learns patterns from large amounts of training data without being explicitly programmed. The more data it sees, the better it gets. Example: a spam filter learns to recognise spam emails from thousands of examples.
Training Data
The dataset used to teach an AI model. If the training data is biased or incomplete, the AI's decisions will also be biased — this is called algorithmic bias.
Classification
An AI task that assigns labels to data. Example: sorting images into "cat" or "dog". The AI looks at features (colour, shape, size) to decide.
Pattern Recognition
AI identifying repeating patterns in data. Used in face recognition, recommendation systems ("people who watched X also watched Y"), and medical diagnosis.
Ethics & Responsible Use of Data
Key Ethical Issues
Privacy — Personal data should only be collected with consent and stored securely.
Bias — AI trained on biased data makes unfair decisions. Diverse, representative data reduces bias.
Transparency — People should be able to understand how AI decisions that affect them are made.
Security — Data must be protected from unauthorised access using passwords, encryption, and access controls.
💡 A key question for any AI system: Who collected the data? Who does it represent? What decisions is it being used to make?
Data Representation
How Computers Store Data
Binary — Computers store all data as 1s and 0s (bits). A bit is the smallest unit of data.
Byte — 8 bits = 1 byte. A single character (like "A") is stored in one byte.
Kilobyte (KB) — 1,024 bytes. A short text message.
Megabyte (MB) — ~1 million bytes. A photo.
Gigabyte (GB) — ~1 billion bytes. A film.
