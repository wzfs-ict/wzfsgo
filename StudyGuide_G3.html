Makers Empire — 3D Modelling
Learn to design 3D objects using shapes, colours, and the building tools inside Makers Empire.
What is 3D Modelling?
Key Idea

3D vs 2D

A 2D shape is flat — like a drawing on paper. A 3D shape has width, height, AND depth — like a real object you can hold. Makers Empire lets you design 3D shapes on a screen!

Key Idea

What is a Designer?

A designer thinks about what a product looks like and how it works. In Makers Empire, you are the designer — you choose the shapes, size, and purpose of your object.

Key Idea

Design Thinking

Good designers ask: Who is this for? What problem does it solve? Before you build in Makers Empire, think about what you want to make and why.

Makers Empire Tools to Know
Shape Library
A collection of basic 3D shapes you can use to start building — like cubes, cylinders, cones, and spheres.
Move
Drag a shape to a new position on the building platform.
Resize
Make a shape bigger or smaller by dragging its handles.
Rotate
Turn a shape so it faces a different direction.
Combine / Group
Put two or more shapes together to create a new, more complex object.
Hole Tool
Cuts a hole or opening into a shape — useful for making containers, windows, or tunnels.
Colour
Choose a colour for each shape — think about what makes your design clear and appealing.
3D Print Ready
A model that has no floating pieces and is solid — ready to be printed by a real 3D printer!
💡
Assessment Tip: Know the difference between the Hole Tool (removes material) and the Shape Tool (adds material). You may be asked which tool you would use for a task.
Shapes to Recognise
3D Shapes
  • 🟦 Cube / Box — 6 flat square faces
  • 🔵 Sphere — perfectly round, like a ball
  • 🔶 Cylinder — round tube, like a tin can
  • 🔺 Cone — round base with a point on top
  • ⬡ Prism — has two matching flat ends
Design Process Steps
  • 1️⃣ Identify — What is the problem?
  • 2️⃣ Design — Sketch your idea on paper
  • 3️⃣ Build — Create it in Makers Empire
  • 4️⃣ Evaluate — Does it work? Is it good?
  • 5️⃣ Improve — Change what doesn't work

⭐ Remember for the Assessment

  • 3D modelling means designing objects that have width, height AND depth.
  • You use the Shape Library to pick basic shapes, then combine them.
  • The Hole Tool removes material from a shape.
  • The design process: Identify → Design → Build → Evaluate → Improve.
  • Good designers always think about who the design is for.

ScratchJr — Coding Basics
Use colourful code blocks to make characters move, talk, dance, and react! In Grade 3 you'll learn the four big coding ideas: Sequence, Loop, Event, and Trigger.
The ScratchJr Screen
Interface

Key Areas

  • 🎬 Stage — Where your characters move and your story plays
  • 🐱 Sprite — A character or object in your project
  • 📦 Block Area — All the coloured code blocks you can use
  • 🧱 Script Area — Where you drag blocks to build your code
  • 🎨 Paint Editor — Draw or change how your sprite looks
Block Colours

What Each Colour Does

  • Yellow Trigger blocks — start your code
  • Blue Motion — move & jump
  • Red Control — loops & wait
  • Purple Looks — grow, shrink, hide
  • Green Sound — play music & pops
  • Orange Messages — send & receive
💡
Remember: Every script must start with a Yellow Trigger block. Without a trigger, your code will never run! The most common trigger is the Green Flag block.

⭐ Remember for the Assessment

  • A Sprite is a character or object in ScratchJr.
  • Code blocks are dragged into the Script Area to build a program.
  • Every script must start with a Yellow trigger block.
  • ScratchJr runs code from left to right, one block at a time.
  • You can have more than one script for the same sprite.
➡️
Sequence
The order in which code blocks run — always one after another, from left to right.
What is Sequence?
Definition

Steps in Order

A sequence is a set of instructions that run one after another, in a specific order. In ScratchJr, blocks connected in a row form a sequence — the first block runs, then the second, then the third.

Real Life Example

Getting Ready for School

Wake up → Get dressed → Eat breakfast → Go to school. If you change the order (eat breakfast THEN wake up?), things go wrong! The same is true in code — order matters.

Sequence in ScratchJr

A simple sequence that makes a cat move right, then jump, then play a sound:

▶ Green Flag → → Move Right → ↑ Jump → ♪ Pop Sound

Each block waits for the one before it to finish before it runs.

Sequence
A set of steps that happen one after another in a specific order.
Script
A group of connected code blocks that run together as a program.
Order
The position of each step — first, second, third, etc. Changing the order changes the result.
🧪
Test yourself: If you swap block 2 and block 3 in a sequence, what changes? In an exam, you may be shown a row of blocks and asked what happens — read them left to right, one at a time.

⭐ Key Facts — Sequence

  • Sequence = steps in order, one at a time.
  • ScratchJr always runs blocks left to right.
  • Changing the order of blocks changes what the program does.
  • A sequence starts with a trigger block (usually the Green Flag).
🔄
Loop
A loop makes part of your code run again and again — so you don't have to repeat the same block over and over.
What is a Loop?
Definition

Repeat, Repeat, Repeat

A loop is a block (or group of blocks) that runs more than once. Instead of writing "Move Right" 10 times, you use a loop to do it automatically. Loops make code shorter and smarter.

Real Life Example

Clapping in Music Class

If a teacher says "clap 4 times," you clap, clap, clap, clap — that's a loop that runs 4 times! If she says "keep clapping until I stop you," that's a forever loop.

Two Types of Loop in ScratchJr
Type 1

Repeat Forever ♾️

Runs the blocks inside it again and again without stopping, until you press the red Stop button.

▶ Flag → ♾ Repeat Forever

Example: A character spinning forever as decoration.

Type 2

Repeat N Times 🔢

Runs the blocks inside a set number of times, then stops and continues with the rest of the script.

▶ Flag → 🔁 Repeat × 3 → ♪ Pop

Example: A character jumping exactly 3 times.

Loop
A block or group of blocks that runs more than once automatically.
Repeat Forever
A loop that keeps running until you stop the program. It never ends on its own.
Repeat N Times
A loop that runs a specific number of times (e.g. 3 times), then stops.
Iteration
One single run of the loop. If a loop runs 5 times, it has 5 iterations.
⚠️
Watch out: If you put a Repeat Forever block in the middle of a script, the blocks after it will never run — because the forever loop never ends! Put forever loops at the end of a script, or use them on their own.

⭐ Key Facts — Loop

  • Loops make blocks run more than once without re-writing them.
  • Repeat Forever — runs until you press Stop.
  • Repeat N Times — runs a set number of times, then continues.
  • Loops save time and make code much shorter.
⚡
Event & Trigger
Events are things that happen in your project. Triggers are the blocks that watch for those events and start your code running when they occur.
What is an Event?
Definition

Something Happens

An event is something that happens that your program can detect — like tapping a sprite, pressing the green flag, or two characters bumping into each other.

Definition

A Trigger Watches & Responds

A trigger is a yellow block at the start of a script. It watches for a specific event, and starts the script running when that event happens. No trigger = code never runs.

ScratchJr Trigger Blocks (Yellow)
Green Flag ▶
The most common trigger. Starts the script when the green flag is tapped to begin the project.
Start on Tap 👆
Starts the script when the player taps (touches) that specific sprite.
Start on Bump 💥
Starts the script when this sprite bumps (touches) another sprite.
Receive Message 📩
Starts the script when a matching message is sent from another sprite. Messages match by colour.
Send Message 📤
Sends a coloured message to any sprite waiting for it with a Receive Message block.
Event Example — A Bump Reaction

When the cat bumps an obstacle, it goes back to the start:

💥 Start on Bump → 🏠 Go Home

The event is the bump. The trigger watches for it. When it happens, "Go Home" runs.

Message Example — Two Sprites Talking

Sprite 1 sends a message when tapped. Sprite 2 receives it and starts dancing:

SPRITE 1 — Cat
👆 On Tap → 📤 Send 🟠
SPRITE 2 — Dog
📩 Receive 🟠 → 🔄 Spin
🔑
Matching Messages: A Send Message and Receive Message only work together if they are the same colour. An orange send only triggers an orange receive. Colours must match!

⭐ Key Facts — Event & Trigger

  • An event is something that happens (tap, bump, message).
  • A trigger is the yellow block that watches for an event and starts the script.
  • Every script needs a trigger at the beginning — otherwise it won't run.
  • Messages must be the same colour to match and work.
  • One sprite can send a message that starts code in a completely different sprite.
