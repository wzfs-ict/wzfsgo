Grade 3

Building Digital Foundations

Mr. Joshua WZFS

Instructions: Read carefully. You must memorize the blue Vocabulary Words. Computers are not smart; they rely on the exact instructions provided by disciplined engineers.

ScratchJr: Sequence

A computer reads instructions exactly as they are written. We must be precise.

Sequence:

Putting code blocks in the exact right order. The computer reads the sequence from left to right, like reading a book.

If your blocks are in the wrong order, the character will do the wrong action. The sequence must be perfect.

ScratchJr: Triggers

Event / Trigger:

The yellow blocks. This is the specific action that tells the computer to start running the code sequence.

Two Main Triggers:

  • Start on Green Flag: Tapping the flag at the top of the screen starts the code.
  • Start on Tap: Touching the character itself with your finger starts the code.

ScratchJr: Efficiency

Engineers do not waste time writing the same code over and over.

Loop:

The orange block. It wraps around a sequence to repeat an action automatically.

Number Parameter:

The number typed on a block. It dictates how many times to loop or how far to move.

ScratchJr: Fixing Errors

Computers make no mistakes. If the program fails, the human engineer wrote bad instructions.

Bug:

An error or mistake in your code sequence that stops the program from working correctly.

Debugging:

The process of finding the mistake and fixing it so the sequence runs perfectly.

Makers Empire: Space

To build digital structures, you must understand physical space.

3D Shape:

A solid object. It has Width, Height, AND Depth. (A 2D shape is flat, like a drawing on paper).

Workplane:

The flat floor or grid on the screen where you place shapes to begin building.

Makers Empire: Tools

Tools allow engineers to shape objects to their exact needs.

Move:

Sliding an object up, down, left, or right across the workplane.

Resize:

Pulling the corners of an object to make it larger or smaller.

Rotate:

Turning an object around in a circle to view or attach different sides.

Grade 4

Animation and Logic

Mr. Joshua WZFS

Instructions: Master the green Vocabulary Words. You will learn the strict rules of programming events and the traditional methods of frame-by-frame animation.

ScratchJr: Strict Logic

Programming requires discipline. If one step is wrong, the system breaks.

Sequence:

A strict list of steps. If a block is out of order, the program fails. This error is called a Bug.

Loop Parameter:

You must calculate the exact number required in the orange loop block to reach your target perfectly, rather than guessing.

ScratchJr: Interactions

How do multiple characters interact?

Start on Bump:

A trigger that starts code ONLY when two characters crash into each other.

Send Message:

The closed envelope block. Sprite A sends a hidden color signal to Sprite B to tell it to start moving.

Algorithm Planning

We never touch the tablet until the design is approved on paper.

Storyboard:

A paper plan showing what characters will do in each scene, drawn step by step.

Algorithm:

A set of rules to solve a problem. Writing down the exact sequence of blocks you need before you code.

Piskel: Stop Motion

Animation is the classic art of displaying still pictures very quickly to fake movement.

Frame:

One single, flat picture. An animation is made of many frames played in sequence.

Duplicate:

To make an exact copy of a frame so you only have to make a tiny change for the next step.

Piskel: Speed & Shadow

How do we control and perfect our animation?

FPS (Frames Per Second):

This number controls the Speed. A high FPS plays the animation fast. A low FPS plays it slow.

Onion Skinning:

A tool that shows a faded ghost of your last frame. You use this shadow to draw the next frame perfectly in line.

Grade 5

Systems & Artificial Intelligence

Mr. Joshua WZFS

Instructions: Master the purple Vocabulary Words. You are transitioning to real-world software engineering, physical robotics, and mathematical design.

ScratchJr: Mastery

You must master visual logic before moving to text-based code.

Sequence:

The strict, literal order of instructions.

Loop:

Repeating code to ensure efficiency and save time.

Event / Trigger:

The action that starts the code. Grade 5 engineers rely on Message Envelopes to sync multiple characters.

Botley: Physical Code

Code doesn't just live on a screen; it controls physical machines.

Physical Computing:

Writing code to control a real object in the real world.

Command:

Pressing buttons on the remote to build a Sequence (Forward, Left).

Transmit:

Sending the programmed instructions invisibly from the remote to the robot.

AI & Logic

Artificial Intelligence uses data to mimic the human brain.

Data:

Information (images, numbers, text) fed to a computer to train it.

Pattern Recognition:

How AI processes huge amounts of data to find similarities (e.g., recognizing a face or a stop sign).

3D Engineering

How do we precisely measure 3D space?

Axes:

The invisible math lines that show direction.

  • X-axis: Left and Right
  • Y-axis: Up and Down
  • Z-axis: Forward and Backward (Depth)
Grouping:

Fusing two different 3D shapes together to form a single solid object.

Vector Graphics

Professional designers use math (Method Draw), not pixels.

Vector:

An image drawn using math equations. It never becomes blurry, even if you make it huge.

Path & Node:

The Path is the line connecting points. The Node (Anchor Point) is the dot you drag to bend the shape.
